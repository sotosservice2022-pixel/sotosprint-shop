ℹ️ Якщо домен куплений у Cloudflare Registrar або вже доданий у твій CF-акаунт як сайт — все автоматично.

🚀 Майстер міграції на новий домен

⚠️ Цей процес не зламає поточний agprnt.com. Він просто:
  1. Знайде Zone ID для нового домена через CF API
  2. Оновить settings.primaryDomain і cfZoneId
  3. Оновить URL Telegram webhook
Старий домен залишається активним доки ти його не видалиш вручну.
Цей домен має бути зі списку «Поточний стан» вище і мати статус active.
«Оновити webhook» корисно якщо при першій спробі переходу DNS ще не встиг пропагуватись (типова помилка «Failed to resolve host»). Зачекайте 5-30 хв після додавання домена і спробуйте.

📚 Інструкції

⚙️ Перед міграцією — налаштуй CF API Token

Щоб все працювало автоматично, у твоєму CF API token (dash.cloudflare.com/profile/api-tokens) повинні бути такі дозволи:

  • Account → Cloudflare Pages → Edit — для додавання custom domain
  • Account → Account Analytics → Read — для віджета статистики
  • Zone → Cache Purge → Purge — для миттєвого оновлення кешу
  • Zone → DNS → Edit — для автоматичного створення CNAME запису

Resources: Account: Include всі + Zone: All zones from an account

Якщо Zone → DNS → Edit відсутнє — CNAME доведеться створити вручну в CF Dashboard → твій домен → DNS → Records → +Add → CNAME, name @, target agprnt-shop.pages.dev, Proxied 🟠

Випадок 1: Домен куплений у Cloudflare Registrar

  1. Купи домен на dash.cloudflare.com → Registrar
  2. Домен автоматично доданий у твій акаунт як сайт
  3. Введи його у поле «Додати новий домен» вище → ➕ Додати
  4. Через 2-5 хв у списку зʼявиться зі статусом active
  5. Введи його у «Майстер міграції» → 🚀 Перейти на цей домен
  6. Готово — сайт доступний на новому домені

Випадок 2: Домен у стороннього регістратора (Namecheap, GoDaddy і т.д.)

  1. Зайди в dash.cloudflare.com → + Add a site
  2. Введи свій домен → Continue → вибери Free план
  3. CF покаже 2 NS-сервери (типу aron.ns.cloudflare.com)
  4. Зайди в кабінет регістратора (Namecheap/GoDaddy/etc) → знайди налаштування DNS свого домену
  5. Зміни NS-сервери на ті що дав CF → збережи
  6. Чекай 5 хв - 24 години (зазвичай 30 хв) поки CF побачить що NS прописані. У CF на сторінці домену зʼявиться «Active»
  7. Тепер так само як у Випадку 1: введи домен у «Додати новий домен» → ➕ Додати → дочекайся active → «Майстер міграції»

Що робити після переходу

  • Зберегти старий з редиректом: Cloudflare Dashboard → Pages → agprnt-shop → Custom domains → старий домен → ⋯ → Set up redirect → вибери новий
  • Видалити старий: тут вище у списку «Поточний стан» → кнопка [ × Видалити ]
  • Telegram webhook: оновився автоматично під час міграції
  • SEO: пошуковики помітять новий домен через 1-2 тижні. Прискорити через Google Search Console.
